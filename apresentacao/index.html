01

O Problema Físico

Um sistema clássico da mecânica newtoniana: três blocos conectados por cordas ideais passando por uma polia sem massa.

m₁

Bloco 1

Sobre o plano inclinado, conectado por corda ao bloco 2. Sujeito a peso, normal, atrito e tração T₁.

m₂

Bloco 2

Também sobre o plano, entre m₁ e a polia. Sujeito a peso, normal, atrito e trações T₁ e T₂.

m₃

Bloco 3

Pendurado verticalmente pela polia. Seu peso gera a força motriz do sistema.

02

Diagrama de Corpo Livre — m₁

Análise isolada das forças que atuam no bloco 1 sobre o plano inclinado.

Peso P₁ = m₁·g

Decomposto em P₁ₓ = m₁·g·sin α (paralelo) e P₁ᵧ = m₁·g·cos α (perpendicular)

Normal N₁ = m₁·g·cos α

Reação do plano, perpendicular à superfície, anula P₁ᵧ

Atrito f₁ = μ·N₁

Opõe-se ao movimento. Usa μₑ (estático) ou μc (cinético)

Tração T₁

Força da corda que conecta m₁ a m₂, ao longo do plano

03

Diagrama de Corpo Livre — m₂

O bloco intermediário recebe duas trações distintas: T₁ (de m₁) e T₂ (da polia para m₃).

Peso P₂ = m₂·g

Componentes P₂ₓ = m₂·g·sin α e P₂ᵧ = m₂·g·cos α

Normal N₂ = m₂·g·cos α

Reação perpendicular do plano inclinado

Atrito f₂ = μ·N₂

Opõe-se ao sentido do movimento relativo

Tração T₁ (rampa abaixo)

Puxado pelo bloco m₁ no sentido da rampa para baixo

Tração T₂ (rampa acima)

Puxado pela corda que passa pela polia, conectada a m₃

04

Diagrama de Corpo Livre — m₃

O bloco suspenso é o mais simples: apenas seu peso e a tração da corda atuam sobre ele.

Peso P₃ = m₃·g

Força gravitacional atuando verticalmente para baixo

Tração T₂ (para cima)

Força da corda que passa pela polia, suspendendo o bloco

2ª Lei de Newton para m₃

P₃ − T₂ = m₃ · a
T₂ = m₃ · (g − a)

Se o sistema está em equilíbrio (a = 0), então T₂ = m₃·g

05

Equações do Sistema

O algoritmo central resolve o sistema em etapas lógicas, verificando primeiro o equilíbrio estático.

1 Teste de Equilíbrio
F_net = P₃ − (P₁ₓ + P₂ₓ) f_max = μₑ · (N₁ + N₂) |F_net| ≤ f_max → Equilíbrio
2 Aceleração (se move)
f_c = μc · (N₁ + N₂) a = (P₃ − P₁ₓ − P₂ₓ ∓ f_c) / (m₁+m₂+m₃) + f_c se desce, − f_c se sobe
3 Trações (blocos sobem)
T₁ = m₁(a + g·sinα + μc·g·cosα) T₂ = m₃(g − a)
4 Trações (blocos descem)
T₁ = m₁(g·sinα − μc·g·cosα − |a|) T₂ = m₃(g + |a|)

function calc() {
    const P1x = m1 * g * Math.sin(alpha);
    const N1  = m1 * g * Math.cos(alpha);
    const Fnet = P3 - (P1x + P2x);

    // Teste de equilíbrio estático
    if (Math.abs(Fnet) <= fT_max) {
        dir = 'Equilíbrio';
        acc = 0;
    } else {
        // Movimento com atrito cinético
        acc = (P3 - P1x - P2x - fT) / (m1+m2+m3);
    }
}

hiDPI Canvas Setup 
function hiDPI(canvas, w, h) {
    const dpr = devicePixelRatio || 1;
    canvas.width  = w * dpr;
    canvas.height = h * dpr;
    canvas.style.width  = w + 'px';
    canvas.style.height = h + 'px';
    const ctx = canvas.getContext('2d');
    ctx.setTransform(dpr,0,0,dpr,0,0);
    return ctx;
}

Integração da Interface

Como os inputs do usuário atualizam o estado do simulador e re-renderizam os resultados em tempo real.

Event Listeners & Atualização 
// Atualiza os valores ao mover os sliders
const inputs = document.querySelectorAll('input[type="range"]');
inputs.forEach(input => {
    input.addEventListener('input', () => {
        // 1. Sincroniza o valor do badge visual
        updateBadge(input);
        
        // 2. Recalcula todas as forças e acelerações
        const results = calc();
        
        // 3. Atualiza as tabelas de força na UI
        updateTables(results);
        
        // 4. Solicita um novo frame no Canvas para fluidez 60fps
        requestAnimationFrame(() => drawMain(results));
    });
});

Arquitetura Reativa: A ausência de frameworks (como React ou Vue) exige que o gerenciamento de estado seja feito manualmente. O evento input garante que os cálculos algébricos rigorosos, a tabela de resultados e o Canvas permaneçam sempre em total sincronia sem necessidade de recarregar a página.
